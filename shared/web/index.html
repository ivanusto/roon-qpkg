⚠️ 為什麼強制使用 host 網路模式？

    

      Roon 透過區網多播（multicast）／廣播來探索串流播放裝置（RAAT 端點、Roon Ready 器材、AirPlay/Chromecast）
      以及遙控器 App。若容器使用 bridge/NAT 網路，多播封包無法跨越 NAT 邊界，
      裝置探索與遙控器連線會完全失效。因此本套件在每次啟動時都會檢查容器是否為
      --net=host，若被改動會自動以 host 模式重建容器。請勿在 Container Station 中修改此設定。
    

  


  

    
💾 資料庫請放在 SSD 儲存池

    

      Roon 資料庫與快取（掛載到容器內的 /Roon）充滿小型隨機讀寫。放在傳統硬碟（HDD）磁碟區時，
      隨機讀寫延遲會造成音訊解碼不順、播放清單與媒體庫載入緩慢、搜尋卡頓。
    

    

      設定方式：編輯 QPKG 安裝目錄下的 roon.conf，將
      ROON_DATA_PATH 指向 SSD 磁碟區上的路徑（例如
      /share/CACHEDEV2_DATA/RoonServer/data），再重新啟動本套件。
    

    
    
      
  1. 「儲存與快照總管」確認哪個磁碟區位於 SSD 儲存池（純 SSD 池，非 SSD 快取）。
    2. 
      
  2. SSH 登入 NAS，編輯 <QPKG 安裝目錄>/roon.conf：
    
        ROON_DATA_PATH="/share/CACHEDEV2_DATA/RoonServer/data"
    3. 
      
  3. 若要搬移既有資料庫：先停止套件，將舊資料夾整個複製到新路徑，再啟動。
    4. 
      
  4. App Center 重新啟動 Roon Server，或執行 /etc/init.d/roon-server-docker.sh restart。
    5. 
    

  


  

    
📁 資料夾設定

    

      直接在此設定掛載路徑，儲存後約 1 分鐘內自動套用並重建容器
      （Roon 資料庫保留；變更資料庫路徑不會自動搬移舊資料）。
      輸入框會提示 NAS 上現有的共享資料夾。
    

    
    
    
    
    
    
    
    

    
    

      在 Roon App 中請用「新增資料夾」選 /Music。
      不要用「新增網路共享位址（SMB）」——Roon 跑在容器內，無法掛載 SMB
      （會出現 UnexpectedError），而且音樂已由本套件直接掛載到 /Music，不需繞道網路。
    

  


  

    
🚀 首次使用

    
    
      
  1. 安裝完成後，套件會於背景呼叫 Container Station 下載官方映像檔
        ghcr.io/roonlabs/roonserver:latest，完成後自動建立並啟動容器（視網速約數分鐘）。
    2. 
      
  2. 在同一區網的電腦／平板安裝 Roon App，
        它會自動找到這台 NAS 上的 Roon Server（host 網路 + 多播探索）。
    3. 
      
  3. 登入 Roon 帳號，於「設定 → 儲存位置」按「新增資料夾」選擇容器內的 /Music（勿使用 SMB 網路共享）。
    4. 
      
  4. 建議在 Roon「設定 → 備份」把備份目的地設為 /RoonBackups。
    5. 
    

    
下載進度：<QPKG 安裝目錄>/logs/pull.log；服務記錄：logs/roon-server.log。

  


  

    
🔧 常用維護指令（SSH）

    
/etc/init.d/roon-server-docker.sh status    # 檢視狀態
/etc/init.d/roon-server-docker.sh restart   # 重新啟動
/etc/init.d/roon-server-docker.sh update    # 更新官方映像並重建容器（資料庫保留）
