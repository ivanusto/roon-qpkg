⚠️ Why is Host Network Mode Enforced?

    

      Roon relies on local network multicast/broadcast (RAAT protocol) to discover audio output devices (Roon Ready, AirPlay, Chromecast) and remote control apps. If the container uses bridge/NAT network, multicast packets are blocked, rendering device discovery and remote control entirely non-functional. Therefore, this app checks for --net=host on every start; if altered, it will automatically recreate the container in host mode. Do not modify this setting in Container Station.
    

  


  

    
💾 Please Keep Database on SSD Storage Pool

    

      The Roon database and cache (mounted as /Roon inside the container) perform heavy small-block random reads/writes. Storing them on a mechanical hard drive (HDD) volume causes high latency, leading to stuttering audio playback, slow playlist/library loading, and sluggish searches.
    

    

      Configuration: Edit roon.conf in the QPKG installation directory and point ROON_DATA_PATH to a path on an SSD volume (e.g. /share/CACHEDEV2_DATA/RoonServer/data), then restart the app.
    

    
    
      
  1. In "Storage & Snapshots Manager", check which volume resides on an SSD storage pool (must be a pure SSD pool; SSD cache does not count).
    2. 
      
  2. Log in to the NAS via SSH and edit <QPKG Root>/roon.conf:
    ROON_DATA_PATH="/share/CACHEDEV2_DATA/RoonServer/data"
    3. 
      
  3. To migrate an existing database: Stop the app first, copy the entire data folder to the new path, and start the app again.
    4. 
      
  4. Restart Roon Server in App Center, or run /etc/init.d/roon-server-docker.sh restart.
    5. 
    

  


  

    
📁 Folder Settings

    

      Configure mount paths directly. Changes are applied automatically and the container is rebuilt within 1 minute (the Roon database is preserved; changing the database path does not automatically migrate existing data). Input boxes will suggest existing shared folders on the NAS.
    

    
    
    
    
    
    
    
    

    
    

      In Roon App, select "Add folder" and choose /Music. Do not use "Add network share (SMB)" — Roon runs inside a container and cannot mount SMB shares (results in UnexpectedError). Your music is already directly mounted to /Music by this app, so no network routing is needed.
    

  


  

    
🚀 Getting Started

    
    
      
  1. Upon installation, the app calls Container Station in the background to download the official image ghcr.io/roonlabs/roonserver:latest. The container is created and started automatically once complete (takes a few minutes depending on network speed).
    2. 
      
  2. Install the Roon App on a computer or tablet connected to the same network. It will discover the Roon Server on this NAS automatically (via host network & multicast discovery).
    3. 
      
  3. Log in to your Roon account, go to "Settings → Storage", click "Add folder", and select /Music inside the container (do not use SMB network sharing).
    4. 
      
  4. It is recommended to set your backup destination to /RoonBackups under "Settings → Backups" in Roon.
    5. 
    

    
Download progress: <QPKG Root>/logs/pull.log; Service log: logs/roon-server.log.

  


  

    
🔧 Useful Maintenance Commands (SSH)

    
/etc/init.d/roon-server-docker.sh status    # Check status
/etc/init.d/roon-server-docker.sh restart   # Restart
/etc/init.d/roon-server-docker.sh update    # Update official image & rebuild container (database is kept)
